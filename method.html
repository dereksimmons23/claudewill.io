claudewill* the method

the method

A practice for getting work done. Five steps. Five principles. Built from 14 months and 100,000+ messages across a dozen real projects.

the session

start

Know where you are before you move.

Read the state file. Check what's live. Name the one priority — not three, one. Say it out loud. Context makes the work better. The intent, not the task list.

work

Do it together.

Set direction, not implementation. Stay present. Course-correct early. You bring judgment, taste, context, lived experience. The machine brings pattern recognition, speed, and no ego about being redirected.

finish

Leave it so tomorrow-you can pick it up cold.

What happened. What was decided. What's next. What's open. Decisions disappear without writing them down. The handoff is the product — not the work itself, but the state you leave behind.

decide

Look from four angles.

Earn — does this pay? Connect — what does this touch? Sharpen — does it meet the standard? Challenge — should you though? Note where the lenses clash. Then make the call. Don't hedge.

rest

Stop.

The other four don't work without this one. Rest is what makes Decide a reconsideration instead of a reflex. It's not a reward for finishing. It's the step that makes the practice sustainable.

the standard

Truth over comfort. Say what's real, not what's easy to hear.
Usefulness over purity. Does it help someone today? Ship it.
Transparency over reputation. Show the work, including where it's ugly.
People over systems. If the process hurts the person, fix the process.
Agency over ideology. Help people think, not tell them what to think.

the files

Two files carry the practice between sessions. Both are plain text. Both work with any tool.

CLAUDE.md

The contract. Identity, voice, boundaries, architecture. What the machine needs to know to do good work with you. Every project has one. It defines how you work, not what you're building.

HANDOFF.md

The baton. Where you are right now. What's next. What's open. Updated at the end of every session. Tomorrow-you reads it cold and knows exactly where to pick up. The relay, not the runner.

the relay

The method came from someone who built things that lasted and never wrote any of it down. His grandson is writing it down — in code, in method, in practice — so it doesn't require him.

The fire doesn't need to be passed. It needs to be carried.
